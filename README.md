# Bank System Simulation (ООП на Python)

Учебный проект, созданный для отработки ключевых принципов объектно-ориентированного программирования в контексте банковских продуктов.

### Основной функционал:
* **Иерархия счетов**: Реализован базовый класс `BankAccount` и специализированные классы `KidsAccount` (с лимитами) и `GoldAccount` (с привилегиями).
* **Система транзакций**: Метод `transfer` позволяет переводить средства между любыми типами аккаунтов с автоматическим расчетом комиссий.
* **История операций**: Каждый объект хранит детальный лог всех действий (пополнения, снятия, переводы, начисление процентов).

### Технические особенности:
1. **Инкапсуляция**: Использование защищенных атрибутов (`_balance`, `_history`) для предотвращения прямого изменения данных извне.
2. **Полиморфизм**: Переопределение метода `withdraw` для реализации разной логики списания средств (лимиты для детей, отсутствие комиссии для Gold-клиентов).
3. **Наследование**: Применение `super().__init__` для эффективного расширения функционала базового класса без дублирования кода.

### Как запустить:
Код находится в файле `Portfolio.py`. Для тестирования достаточно запустить скрипт, в нижней части которого уже прописаны сценарии взаимодействия между клиентами (Артем, Петр, Иван).
